## Honest Feedback

**Friction 1: Context overflow recovery is imprecise.** The summary from the previous session was good but not perfect. I had to re-verify task statuses because I couldn't be 100% certain what was actually committed vs what was in-progress. A more structured handoff (like a checkpoint file listing completed tasks with commit hashes) would make recovery instant.

**Friction 2: Agent completion detection is awkward.** I had to poll output files with tail/grep to check if agents finished, then parse JSONL to find results. The notification system works but there's no clean way to know "agent is done, here's the one-line summary." I ended up reading raw JSONL and guessing from message counts.

**Friction 3: TSC running from wrong CWD.** Multiple times `npx tsc` ran from `/Users/lambogreny/oracles/frontend` (the parent repo) instead of `/Users/lambogreny/oracles/smsok-clone`, causing confusing module resolution errors. The shell CWD keeps resetting. I should always use `cd /path && npx tsc` as a single command.

## Lessons Learned

1. **Promise.allSettled for parallel fetches** — better than Promise.all when individual API failures should degrade gracefully, not crash the whole page
2. **Mock data audit before claiming "done"** — QA found mock data we missed because we only searched for specific patterns. A comprehensive grep for MOCK_, generateMock, hardcoded arrays would catch more
3. **Verify before marking done** — always run `grep` to confirm zero remaining violations before marking a terminology/cleanup task complete
